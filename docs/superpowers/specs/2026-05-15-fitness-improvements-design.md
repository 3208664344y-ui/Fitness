# 健身日历改进设计

## 总览

两项改动：修复组数切换抖动 + 动作分类系统改造。

## 1. 修复组数切换布局抖动

**原因：** 弹窗为底部弹出式（bottom sheet），高度跟随内容自适应。组数从 3 变到 5 时内容变高，弹窗往上顶；从 5 变回 3 时内容变矮，弹窗往下缩。视觉上表现为"上下抖动"。

**修复：**
- `.sheet` 添加 `min-height: ~420px`，确保弹窗在组数变化时高度不变
- `.set-detail-list` 区域添加 `min-height: 180px`，内部内容变化不影响外部
- 组数超出可视区域时内部滚动，不撑开弹窗

影响范围：`fitness.html:166-176`（sheet 样式）、`:197-215`（set-detail-row 样式）。

### 弹窗交互流程

1. 弹窗打开 → 自定义输入框上方新增「分类标签栏」和「动作网格」
2. 分类标签栏：横向可滚动，显示所有分组名，默认选中第一个
3. 动作网格：2 列按钮排列当前分组下的动作，点击选中
4. 选中的动作名同步到自定义输入框，高亮对应按钮
5. 新增"自定义"分类标签（排在最后），点击后隐藏网格、显示文本输入框让用户自由输入

### 样式要点

- 标签栏：`overflow-x: auto; white-space: nowrap`，水平滚动
- 动作按钮网格：`grid-template-columns: 1fr 1fr; gap: 8px`
- 选中状态：蓝色背景 + 白色文字（与现有 `.selected` 风格一致）
- 网格区域固定高度 `max-height: 200px; overflow-y: auto`，防止撑开弹窗

### 与现有功能的对接

- 编辑模式：打开弹窗时根据 `entry.name` 自动定位到对应分组并高亮
- 自定义输入框：保留，当选中"自定义"标签或直接输入时使用
- `PRESETS` 兼容：改为从 `CATEGORIES` 扁平生成，保持 `sel.value` 校验逻辑不变
