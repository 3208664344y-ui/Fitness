# 训练计划与实时训练模式设计

## 总览

新增训练计划和实时训练模式，让用户提前制定训练模板，训练时按计划逐步执行并自动记录。

## 1. 数据结构

### 训练计划 (`fitness_plans`)

```js
// localStorage key: 'fitness_plans'
// 值: 对象, key 为 plan id
{
  "plan_xxx": {
    id: "plan_xxx",
    name: "推胸日",
    exercises: [
      { name: "卧推", sets: 4, targetWeight: 60, targetReps: 8 },
      { name: "上斜哑铃卧推", sets: 3, targetWeight: 24, targetReps: 10 }
    ],
    createdAt: "2026-05-15",
    lastUsed: "2026-05-12"
  }
}
```

### 训练会话（内存）

```js
session = {
  planId: "plan_xxx",
  currentExIndex: 0,
  currentSetIndex: 0,
  completedSets: [],
  paused: false
}
```

## 2. 导航结构

日历主页面添加底部 Tab 栏：「日历」|「训练」

点击「训练」→ 显示训练计划列表（`#trainingView`）

三个子视图（均在 `#trainingView` 内切换）：
- **计划列表**（默认）
- **计划编辑**
- **训练执行**

## 3. 计划列表页

- 标题「训练」+ 右上角「＋ 新建」按钮
- 计划卡片：名称、动作数、上次使用日期
- 点击卡片 → 进入训练执行
- 长按/右滑卡片 → 编辑/删除
- 空状态提示

## 4. 计划编辑页

- 计划名称输入
- 动作列表（每个动作显示：名称、组数）
- 「添加动作」按钮 → 复用现有分类选择 + 目标重量/次数设置
- 每个动作可编辑/删除/上下移动
- 保存按钮

## 5. 训练执行页

- 顶部栏：计划名 | 暂停按钮 | 进度 "2/4"
- 动作进度圆点行：完成=蓝底✓ | 当前=蓝边框 | 待做=灰
- 当前动作名称（大字）
- 当前组信息："第 2 组（共 4 组）· 目标 60kg × 8"
- 重量/次数大字显示区，点击弹出键盘
- 组类型切换（正式/热身/递减）
- 「完成本组」→ 记录当前组，跳到下一组/动作
- 「跳过」→ 跳过当前组
- 全屏自动保存到当天日历

## 6. 训练完成后

- 展示统计：动作数、组数、总容量
- 「返回日历」按钮
- 数据已自动保存到 selectedDate 对应的日历数据中
